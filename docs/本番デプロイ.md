# 本番環境のしかた（Vercel + LINE + Supabase）

ローカル（`npm run dev`）は使わず、**本番だけ**で動かすためのメモです。

---

## 1. Vercel にデプロイする

1. [Vercel](https://vercel.com/) にログイン
2. このリポジトリ（`diet-ai-app`）をインポート
3. **Root Directory** が `diet-ai-app` ならそのまま。リポジトリ直下がプロジェクトならそのまま
4. デプロイが終わると URL ができる（例: `https://xxxx.vercel.app`）

---

## 2. Vercel に入れる環境変数（全部ここでOK）

**Settings → Environment Variables** に、次を **Production**（必要なら Preview も）に追加します。

| 名前 | なに用？ |
|------|----------|
| `LINE_CHANNEL_SECRET` | LINE Developers のチャネルシークレット |
| `LINE_CHANNEL_ACCESS_TOKEN` | LINE のチャネルアクセストークン（長いやつ） |
| `OPENAI_API_KEY` | OpenAI の API キー |
| `SUPABASE_URL` | Supabase プロジェクトの URL |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase の service_role キー（**絶対に公開しない**） |
| `REMIND_SECRET` | （リマインドを使うときだけ）`/api/remind` 用の合言葉 |
| `CRON_SECRET` | （Vercel Cron を使うときだけ）`REMIND_SECRET` と同じでも可 |

**いまリマインドはオフ**なら、`REMIND_SECRET` / `CRON_SECRET` は未設定でも他機能は動きます。

変数を追加したあと **Redeploy**（再デプロイ）しないと、古い設定のまま動くことがあります。

---

## 3. LINE の Webhook を本番 URL にする

1. [LINE Developers](https://developers.line.biz/) → チャネル → **Messaging API**
2. **Webhook URL** を  
   `https://あなたのドメイン.vercel.app/api/webhook`  
   にする
3. **Webhook の利用** をオン
4. **検証** が成功するか確認

---

## 4. 生理リマインド（※いまはオフ想定）

**定期リマインドは一旦やめている**場合のメモです。

### 止めるときにやること

1. **Vercel**  
   - ダッシュボード → プロジェクト → **Cron Jobs** にジョブが残っていたら **削除**  
   - （リポジトリからは `vercel.json` を削除済み。再デプロイしても Git からは Cron が増えません）

2. **Supabase**（以前 `004` で pg_cron を入れたことがある人だけ）  
   - SQL Editor で次を実行:  
     `select cron.unschedule('daily-period-remind');`  
   - または `supabase/sql/007_disable_supabase_cron_if_using_vercel.sql` を実行

### あとでまた付けたいとき

- **Vercel Cron:** リポジトリに `vercel.json` の `crons` を戻す + `CRON_SECRET` を設定  
- **Supabase:** `004` の `cron.schedule` 部分を URL・合言葉ごと合わせて再実行（または手動で `cron.schedule`）

---

## 5. 本番でリマインドが動いたか試す（リマインドを使うときだけ）

デプロイした URL で、**合言葉**を付けて開きます（`REMIND_SECRET` と同じ）:

```
https://あなたのドメイン.vercel.app/api/remind?secret=あなたのREMIND_SECRET
```

- JSON で `ok: true` が返れば **認証OK**
- `sentCount` は「今日の条件でLINEに送った人数」（0でもエラーではないことが多い）

---

## 6. よくあるつまずき

| 现象 | 対処 |
|------|------|
| Webhook 検証失敗 | URL の typo、`/api/webhook` まで含むか |
| リマインド 401 | `REMIND_SECRET` / `CRON_SECRET` が Vercel に無い、または URL の `secret=` が違う |
| リマインド 503（秘密なし） | 環境変数を入れたあと **再デプロイ** |
| リマインドは通るが LINE が来ない | 今日が「送る日」じゃない、または `menstrual_cycles` のデータがない |

---

## 7. ローカルについて

本番だけ使うなら、**ローカル用の `.env.local` は必須ではありません。**  
全部 Vercel の Environment Variables に揃えれば動きます。
